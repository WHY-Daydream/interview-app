# AI 大模型开发工程师 面试题总结 - 2026-05-22

## 基础题

**Q1: 请解释 PyTorch 中 Tensor 与 NumPy 数组的主要区别，并说明如何在两者之间进行高效转换？**
**难度**: 低  
**关联知识点**: PyTorch、Tensor、NumPy  
**答案**: 
Tensor 是 PyTorch 的核心数据结构，与 NumPy 数组类似，但支持 GPU 加速和自动微分。主要区别包括：(1) Tensor 可以在 GPU 上运行，而 NumPy 数组只能在 CPU 上运行；(2) Tensor 支持自动梯度计算（requires_grad）；(3) Tensor 是动态计算图的基础。转换方法：使用 `Tensor.numpy()` 将 Tensor 转为 NumPy 数组，使用 `torch.from_numpy()` 将 NumPy 数组转为 Tensor。注意：这两个转换共享内存，修改一方会影响另一方；若需独立副本，需使用 `.clone()` 或 `.copy()`。

**Q2: 什么是 PyTorch 的动态计算图？它与 TensorFlow 的静态图有何不同？**
**难度**: 中  
**关联知识点**: PyTorch、动态图、静态图  
**答案**: 
动态计算图（Define-by-Run）在每次前向传播时动态构建计算图，图的结构随代码执行而变化，因此更灵活、易于调试和扩展。静态图（Define-and-Run）先定义整个计算图再执行，图结构固定，优化更高效但灵活性较差。PyTorch 的动态图优势包括：支持条件分支和循环等动态控制流、调试更直观（可逐行执行）、模型结构可动态调整。TensorFlow 2.x 已引入 Eager Execution 支持动态模式，但静态图仍是其核心优势。

**Q3: 请列举至少三种创建 PyTorch Tensor 的方法，并说明它们的适用场景。**
**难度**: 低  
**关联知识点**: PyTorch、Tensor 创建  
**答案**: 
(1) `torch.tensor(data)`：从现有数据（列表、NumPy 数组等）创建，适用于已有数据需要转换为 Tensor 的场景；(2) `torch.randn(shape)`：创建服从标准正态分布的随机 Tensor，常用于初始化权重或生成随机数据；(3) `torch.ones(shape)` / `torch.zeros(shape)`：创建全 1 或全 0 的 Tensor，适用于初始化偏置或占位符；(4) `torch.arange(start, end, step)`：创建等差数列 Tensor，适用于生成索引或循环变量；(5) `torch.linspace(start, end, steps)`：创建等间隔序列，适用于生成采样点。

**Q4: 解释 PyTorch 中 `backward()` 方法的作用，以及如何访问计算得到的梯度？**
**难度**: 中  
**关联知识点**: PyTorch、自动微分、Autograd  
**答案**: 
`backward()` 方法用于自动计算计算图中所有叶子节点（requires_grad=True）的梯度，实现反向传播。对于标量输出，直接调用 `loss.backward()`；对于非标量输出，需传入与输出相同形状的梯度权重。梯度通过 `.grad` 属性访问，例如 `x.grad` 返回 x 的梯度值。注意：梯度会累积，每次反向传播前需调用 `optimizer.zero_grad()` 或手动清空梯度，否则梯度会累加。

## 进阶与项目题

**Q5: 如何在 PyTorch 中实现自定义数据集和数据加载？请结合具体代码说明。**
**难度**: 中  
**关联知识点**: PyTorch、DataLoader、Dataset  
**答案**: 
在 PyTorch 中，通过继承 `torch.utils.data.Dataset` 类实现自定义数据集，需重写 `__len__` 和 `__getitem__` 方法。示例：
```python
from torch.utils.data import Dataset, DataLoader
class MyDataset(Dataset):
    def __init__(self, data, labels):
        self.data = data
        self.labels = labels
    def __len__(self):
        return len(self.data)
    def __getitem__(self, idx):
        return self.data[idx], self.labels[idx]
dataset = MyDataset(data, labels)
dataloader = DataLoader(dataset, batch_size=32, shuffle=True, num_workers=4)
```
`DataLoader` 支持批处理、打乱、多线程加载等，需注意 `num_workers` 在 Windows 下可能需设置为 0 以避免多进程问题。

**Q6: 请解释 PyTorch 中 `torch.nn.Module` 的作用，并说明如何定义和训练一个简单的神经网络。**
**难度**: 中  
**关联知识点**: PyTorch、神经网络、nn.Module  
**答案**: 
`torch.nn.Module` 是所有神经网络模块的基类，提供参数管理、模型保存/加载、设备迁移等功能。定义网络需继承 `nn.Module` 并实现 `forward` 方法。示例：
```python
import torch.nn as nn
class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.relu = nn.ReLU()
        self.fc2 = nn.Linear(256, 10)
    def forward(self, x):
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        return x
```
训练流程：定义模型、损失函数（如 `nn.CrossEntropyLoss`）、优化器（如 `torch.optim.Adam`），循环迭代数据、前向传播、计算损失、反向传播、参数更新。注意 `model.train()` 和 `model.eval()` 模式切换。

**Q7: 在 PyTorch 中，如何将一个模型部署到 GPU 上运行？请说明常见注意事项。**
**难度**: 中  
**关联知识点**: PyTorch、GPU、CUDA  
**答案**: 
使用 `.to(device)` 方法将模型和数据迁移到 GPU，其中 `device = torch.device("cuda" if torch.cuda.is_available() else "cpu")`。注意事项：(1) 确保 PyTorch 版本与 CUDA 版本匹配，使用 `torch.cuda.is_available()` 检查 GPU 可用性；(2) 模型和数据需在同一设备上，否则会报错；(3) 多 GPU 训练时使用 `nn.DataParallel` 或 `DistributedDataParallel`；(4) 避免在 GPU 上频繁创建小 Tensor，会增加内存碎片；(5) 训练完成后使用 `model.cpu()` 将模型移回 CPU 进行保存。

**Q8: 请解释 PyTorch 中 `torch.cat` 和 `torch.stack` 的区别，并举例说明各自的应用场景。**
**难度**: 中  
**关联知识点**: PyTorch、张量操作、拼接  
**答案**: 
`torch.cat` 在现有维度上进行拼接，不增加新维度，要求除拼接维度外其他维度相同。例如 `torch.cat([a, b], dim=0)` 将两个形状为 (3,4) 的张量拼接为 (6,4)。`torch.stack` 在新维度上进行堆叠，会创建新维度，要求拼接的张量形状完全相同。例如 `torch.stack([a, b], dim=0)` 将两个形状为 (3,4) 的张量堆叠为 (2,3,4)。应用场景：`cat` 用于批量数据的简单拼接，如训练数据合并；`stack` 用于创建批次维度，如将多个独立样本堆叠为一个批次。

**Q9: 在 PyTorch 中，如何实现梯度裁剪（Gradient Clipping）？它有什么作用？**
**难度**: 高  
**关联知识点**: PyTorch、梯度裁剪、训练优化  
**答案**: 
梯度裁剪通过限制梯度的最大值或范数来防止梯度爆炸。PyTorch 提供两种方法：(1) `torch.nn.utils.clip_grad_norm_(parameters, max_norm)`：根据所有参数的 L2 范数裁剪梯度；(2) `torch.nn.utils.clip_grad_value_(parameters, clip_value)`：将梯度值限制在 [-clip_value, clip_value] 范围内。在反向传播后、优化器更新前调用，例如：
```python
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```
梯度裁剪常用于 RNN、LSTM 等序列模型，能提升训练稳定性。

**Q10: 请结合 PyTorch 的 Autograd 机制，解释为什么需要调用 `optimizer.zero_grad()`？如果不调用会怎样？**
**难度**: 高  
**关联知识点**: PyTorch、Autograd、优化器  
**答案**: 
PyTorch 的 Autograd 机制在每次 `backward()` 时会将梯度累加到 `.grad` 属性上，而不是覆盖。这是因为某些场景（如 RNN 的 BPTT、小批量梯度累积）需要累加梯度。如果不调用 `zero_grad()`，每次反向传播的梯度会累加，导致梯度值过大，破坏参数更新。例如，第一次反向传播后 `x.grad = g1`，第二次反向传播后 `x.grad = g1 + g2`，最终优化器会使用累加后的梯度更新参数，导致训练不稳定。因此，每个训练迭代的标准流程是：`optimizer.zero_grad()` → `loss.backward()` → `optimizer.step()`。

**Q11: 在开发大模型应用（如 Stable Diffusion、YOLOv5）时，PyTorch 的哪些特性对模型部署至关重要？请举例说明。**
**难度**: 高  
**关联知识点**: PyTorch、模型部署、大模型  
**答案**: 
关键特性包括：(1) **TorchScript**：通过 `torch.jit.script` 或 `torch.jit.trace` 将模型序列化为可独立运行的计算图，支持 C++ 部署，提升推理性能；(2) **量化（Quantization）**：使用 `torch.quantization` 将模型权重从 FP32 转为 INT8，减少内存占用和加速推理；(3) **ONNX 导出**：通过 `torch.onnx.export` 将模型转为 ONNX 格式，支持跨框架部署（如 TensorRT）；(4) **混合精度训练**：使用 `torch.cuda.amp` 的自动混合精度（AMP）减少显存占用并加速训练；(5) **分布式训练**：`torch.distributed` 支持多 GPU、多节点训练，适用于大模型。例如，Stable Diffusion 部署时常用 TorchScript 优化推理管线，YOLOv5 常导出为 ONNX 或 TensorRT 格式。

**Q12: 请说明 PyTorch 环境配置中常见的版本兼容性问题，以及如何排查和解决？**
**难度**: 中  
**关联知识点**: PyTorch、环境配置、CUDA  
**答案**: 
常见问题：(1) PyTorch 与 CUDA 版本不匹配：例如 PyTorch 1.10 要求 CUDA 11.1+，安装低版本 CUDA 会导致 GPU 不可用；(2) torchvision 与 PyTorch 版本不配套：官方有严格版本对应表，不匹配可能导致导入失败或运行时错误；(3) cuDNN 版本不兼容：cuDNN 需与 CUDA 版本对应，错误配置会导致训练速度慢或崩溃。排查方法：使用 `torch.__version__` 和 `torch.version.cuda` 查看版本；使用 `torch.cuda.is_available()` 检查 GPU 可用性；查看官方版本兼容性表格。解决方案：推荐使用 conda 创建虚拟环境，通过 `conda install pytorch torchvision cudatoolkit=11.1 -c pytorch` 安装配套版本；或使用 pip 直接从 PyTorch 官网选择对应命令安装。
